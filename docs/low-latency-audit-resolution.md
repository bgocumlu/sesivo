# Low-latency audit resolution map

This maps the findings maintained in
`archive/md-artifacts/root/LOW_LATENCY_AUDIT_2026-07-14.md` to the current implementation and evidence.
“Real gate” means the scenario requires physical hardware or a real network. Those gates are
marked `NOT RUN` and block production readiness until measured evidence passes.

| Finding | Resolution |
|---|---|
| Callback buffers above 960 frames were truncated | Full callbacks are processed in bounded 960-frame chunks. Synthetic 960/1024/2048 continuity tests require non-zero output with no zero tail. |
| Serialized server receive/fan-out | Eight receive operations remain outstanding and each slot is rearmed before validation. Four fixed SPSC worker shards fan out immutable room snapshots on separate cores. The full 32-sender envelope passes at 2.5/5/10/20 ms. |
| Stale backpressure and unbounded fan-out storage | Media uses a fixed 128-buffer aggregate pool, a 10 ms send deadline, separate pool/expiry/send-error counters, non-blocking sends, and 512 KiB kernel buffers. |
| Burst- and participant-dependent callback work | Callback decode work has a hard 256-decode ceiling and reports budget exhaustion. Per-callback sender notification was removed, Windows scheduling uses Pro Audio MMCSS, and callback duration/deadline counters remain available. Physical worst-case traces are a real gate. |
| Client receive serialized rearm and allocation | Eight fixed receive slots replace per-datagram `shared_ptr` allocation. A successful slot is copied and rearmed before endpoint validation, decrypt, and queue admission. |
| No congestion controller | Sender-ingress stats and authenticated receiver delivery reports now cover both upstream and downstream media loss using only resolved delivered/lost playout outcomes, non-applying first-report/epoch baselines, one-use generated-media credit across epochs, and an expiring lower median across qualified receivers. Loss/RTT feedback lowers CBR bitrate, disables redundancy under severe congestion, permits redundancy only inside the original bitrate budget, and cautiously recovers after healthy intervals. Sustained same-path impairment increases 2.5 ms packets to 5 ms; sustained recovery restores the selected preset. Receiver-owned queue catch-up drops skip directly to current media instead of adding PLC latency. |
| App metric was mislabeled end-to-end | UI wording is now “App path,” not E2E. Partial PCM chunks created by fractional resampling are frame-weighted so the displayed average represents played audio rather than chunk count. Device-to-device electrical/acoustic thresholds and required raw evidence are defined separately as a real hardware gate. |
| Clock/drift estimation was heuristic | Clock offset needs four accepted samples, rejects high-RTT outliers, and retains uncertainty. Controlled device sample-clock drift remains a distinct real-hardware gate. |
| Network/device interruption recovery was slow/manual | Ping detection is 1 second, rebind cooldown is 3 seconds, stale participants/clients expire in 5 seconds, and an unexpectedly stopped audio stream reopens with 1/2/4-second bounded backoff. Physical cut/rebind/hot-plug rows are real gates. |
| Pre-auth state could grow without bound | All endpoint buckets share a 4,096-entry expiring LRU with coarse global unknown-source admission. A one-million-source self-test proves the cardinality ceiling and expiry. |
| Metrics could block media and lacked relay timing | A two-slot non-blocking async exporter drops stale snapshots rather than waiting. Metrics v2 includes receive-handler and relay-dwell histograms plus pool, expiry, and send errors. A stalled-writer test proves producer progress. |
| Backend/scheduler support was ambiguous | The operating envelope names the certified backend. Runtime logs API, actual buffer/rate/device latencies and warns on degraded APIs; Windows callback and sender threads use MMCSS. The build reports enabled ASIO/JACK support. |
| Real-time primitives/copies were unproven | The callback no longer notifies a condition variable, all callback chunks and decode bursts are bounded, and callback deadline/max counters expose misses. Allocator/lock/page-fault proof on each runtime remains a real hardware profiler gate. |
| Tests/CI did not establish media behavior | A signed-session multi-client relay harness and full capacity sweep were added. CI runs the relay smoke, ASan/UBSan tests, and a libFuzzer network-parser target. Hardware impairment, acoustic, and soak tiers are enumerated in the real validation record. |
| Direct UDP/configuration was not hardened | Joins always require signed tickets; a standalone server generates an ephemeral 256-bit signing key by default, while persistent/shared deployments can load one from a permission-restricted file. Unknown and removed insecure options fail, plaintext outer audio is rejected, only authenticated room snapshots receive media, and authenticated secure-control forwarding has a dedicated token bucket. Direct-UDP/NAT/firewall limitations and deployment controls are explicit. |
| Any participant could replace a peer's media key | Room creation commits to the media secret without revealing it. Join tickets bind that commitment to the room epoch; the relay forwards key envelopes and secure controls only for the current server-authorized epoch/commitment, and clients reject secrets whose SHA-256 commitment does not match. Only an admin-token-authorized room change can install the next commitment. |
| Readiness/drain/runbook work was absent | SIGINT/SIGTERM withdraw receive admission, export a final metrics snapshot, close UDP, and drain the bounded exporter. Operating-envelope documentation defines readiness, resource, logging, firewall, and restart expectations. |
